# LinkedList

LinkedList는 ArrayList과 유사하다. 둘 다 List 인터페이스를 구현하며, 순서가 있는 데이터의 집합을 다룬다. 하지만 내부 구조와 성능 특성에서 차이가 있다.

## ArrayList vs LinkedList
`LinkedList`는 `ArrayList`과 같이, 동일한 타입의 객체를 저장하는 자료구조다. 메서드 또한 유사하다(둘 다 `List` 인터페이스를 따르기 때문). 즉, `add()`, `get()`, `remove()` 등과 같은 메서드를 사용할 수 있다. 그러나, 둘은 매우 다르게 구축되었다.

ArrayList는 내부에 일반 배열이 존재한다. 요소가 추가되면, 그것은 배열에 배치된다. 배열이 요소를 담기에 충분하지 않으면, 더 큰 배열이 생성되고 기존 배열을 대체한다. 그리고, 기존 배열은 삭제된다.

LinkedList는 요소들을 "컨테이너"에 저장한다. 리스트는 첫 컨테이너에 대한 참조를 유지한다. 각 컨테이너는 요소와 다음 컨테이너에 대한 참조를 포함한다. 마지막 컨테이너는 null을 가리킨다. 요소를 추가할 때, 해당 요소는 새 컨테이너에 저장되고, 해당 컨테이너는 리스트의 다른 컨테이너에 연결된다.

## LinkedList 메서드
| 메서드 | 설명 |
|------|-----|
|`addFirst()`|요소를 리스트의 처음 위치에 추가|
|`addLast()`|요소를 리스트의 마지막 위치에 추가|
|`removeFirst()`|리스트의 첫 번째 요소를 제거하고 반환|
|`removeLast()`|리스트의 마지막 요소를 제거하고 반환|
|`getFirst()`|리스트의 첫 번째 요소를 반환|
|`getLast()`|리스트의 마지막 요소를 반환|

## `var` 키워드
Java 10부터 도입된 `var` 키워드는 지역 변수의 타입을 컴파일러가 자동으로 추론하도록 한다. 이를 통해 코드가 더 간결해질 수 있다. 하지만, 아직 많은 개발자들이 명시적인 타입 선언을 선호하기도 한다.
```java
var cars = new LinkedList<String>();
```
